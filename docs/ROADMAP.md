# 产品路线图（Roadmap）

本文档用于描述扩列社交小程序的后续功能规划，帮助产品、研发、运营按阶段对齐目标。

## 1. 规划原则
- 先做“匹配-开聊-留存”核心闭环，再扩展场景。
- 每个阶段都包含“增长 + 安全 + 体验”三类目标。
- 新功能优先围绕扩列核心场景：快速认识新朋友、低门槛破冰、持续互动。

## 2. 阶段规划

### 2.1 Phase 1（当前迭代）
- 实时聊天升级：补齐离线推送任务队列与弱网下顺序一致性。
- 推送通道升级：已补齐任务消费 Worker、指数退避重试、死信状态与微信订阅消息正式投递通道。
- 推荐体验升级：卡片滑动、推荐理由、冷启动策略优化。
- 举报链路升级：举报入口前置、处理结果反馈、审核处理 SLA。

### 2.2 Phase 2（场景扩展）
- 语音破冰：限时语音速配房。
- 动态广场：发布动态、评论、点赞、话题聚合。
- 兴趣圈子：按标签进圈、圈内群聊、活动帖子。

### 2.3 Phase 3（增长与商业化）
- 会员权益：优先推荐、专属标识、高级筛选。
- 任务体系：签到、邀请、活跃任务。
- 运营平台：活动配置、用户分层、人群触达工具。

## 3. 非功能演进
- 安全：接入内容审核服务、反骚扰策略、风控模型。
- 数据：全链路埋点、留存漏斗、推荐效果看板。
- 工程：测试自动化、灰度发布、告警与可观测性建设。
