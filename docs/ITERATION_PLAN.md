# 迭代执行计划（按周）

本文档将路线图拆解为可执行任务，默认按 6 周推进，可按团队人力压缩为 4 周。

## 1. 里程碑目标
- M1（第 2 周末）：MVP 全链路可跑通（登录-推荐-匹配-聊天-举报）。
- M2（第 4 周末）：实时通信与基础风控上线，推荐体验优化。
- M3（第 6 周末）：语音破冰与动态/圈子一期完成，具备增长试验能力。

## 2. 第 1 周：MVP 基线固化

### 小程序端
- 登录页接入真实微信 `code` 获取逻辑。
- 推荐页、会话页、我的页补齐接口错误态与空态。
- 资料页增加标签选择组件与提交校验。

### 后端 API
- 补齐请求参数校验与统一错误返回结构。
- 增加消息分页参数（会话消息拉取）。
- 增加接口日志字段（traceId、userId、耗时）。

### 数据库
- 确认 `users/swipes/matches/messages/reports` 索引。
- 补齐迁移规范模板（新增 `sql/migrations` 目录约定）。

### 测试与验收
- 通过 1 条真实链路回归：注册->互赞->发消息->举报->后台处理。
- 验收标准：核心 API 成功率 > 99%，P95 推荐接口 < 500ms。

## 3. 第 2 周：管理闭环与上线准备

### 小程序端
- 举报与拉黑入口统一到聊天页和个人详情页。
- 增加封禁状态提示页（被封用户不可继续操作）。

### 后端 API
- 管理接口补充分页、筛选字段（时间区间、处理人）。
- 举报处理结果写回消息/用户可见状态。

### 管理后台
- 举报列表增加处理备注展示。
- 用户管理支持按状态筛选。

### 测试与验收
- 完成回归用例集（功能/权限/异常流）。
- 验收标准：后台处理举报到闭环 <= 24h（人工流程）。

## 4. 第 3 周：实时聊天与在线状态（P1）

### 小程序端
- 接入 WebSocket 长连接。
- 会话页增加未读数、在线状态、输入中提示。

### 后端 API/服务
- 新增实时消息网关服务（连接鉴权、心跳、重连）。
- 增加消息已读回执接口与未读统计接口。

### 数据库
- 新增消息状态字段（sent/delivered/read）。

### 测试与验收
- 断网重连、重复消息、多端登录冲突测试。
- 验收标准：消息到达成功率 > 99.5%，平均到达延迟 < 500ms。

## 5. 第 4 周：推荐体验升级（P1）

### 小程序端
- 推荐页改为卡片滑动交互（like/pass/super-like 可选）。
- 增加“推荐理由”标签（同城、兴趣重合、活跃度）。

### 后端 API
- 推荐接口支持分层策略（冷启动池、活跃池、召回池）。
- 增加去重与频次控制策略。

### 数据与埋点
- 接入关键埋点：曝光、滑动、匹配、首聊。
- 输出首版推荐效果报表字段。

### 测试与验收
- 验收标准：匹配后 24h 首条消息率相较基线提升 >= 10%。

## 6. 第 5 周：语音破冰（P2）

### 小程序端
- 新增“语音破冰房”入口与房间页（计时、静音、退出）。

### 后端服务
- 新增语音房间匹配接口与房间生命周期管理。
- 接入 RTC 服务（厂商 SDK 选型后落地）。

### 风控
- 增加语音房举报与黑名单联动。

### 测试与验收
- 验收标准：语音房接通成功率 > 98%，异常退出可自动恢复。

## 7. 第 6 周：动态广场与圈子一期（P2）

### 小程序端
- 动态列表、发布页、详情页（评论/点赞）。
- 圈子列表与圈子详情页。

### 后端 API
- 动态 CRUD、评论、点赞接口。
- 圈子加入/退出、圈子内容列表接口。

### 运营能力
- 管理台增加动态审核入口与违规下架操作。

### 测试与验收
- 验收标准：内容审核闭环可追溯，违规内容可在 5 分钟内下架。

## 8. 跨周持续任务
- 安全：敏感词、反骚扰策略、封禁复核流程。
- 工程：CI 构建、单测覆盖、日志告警、备份策略。
- 数据：看板建设（留存、匹配率、首聊率、举报率）。

## 9. 人员建议配置（最小）
- 前端（小程序）：2 人
- 后端（API/实时服务）：2 人
- 测试：1 人
- 产品/设计：1 人（可兼职）
- 运营审核：1 人（MVP 可兼职）
