# 扩列社交小程序 Design System v1

## 1. 视觉方向
- 风格：深色霓虹 + 玻璃卡片
- 关键词：科技感、社交氛围、轻动效
- 目标：在小程序内形成可复用视觉规范，减少页面样式分叉

## 2. 设计 Token（落地在 `miniprogram/app.wxss`）
- 背景：深色径向渐变 + 网格纹理
- 文字层级：
  - 主文案：`#ebf2ff`
  - 次文案：`#9eabc9`（`.muted`）
- 品牌主色：青蓝渐变（主按钮、强调标签）
- 危险色：橙红渐变（风险操作）
- 圆角体系：`12rpx`（输入）/ `20rpx`（卡片）/ `24rpx`（Hero）
- 阴影：统一卡片阴影，强调层级与可点击区域

## 3. 组件规范
- 容器：`.container`
- Hero：`.hero` + `.hero-title` + `.hero-subtitle`
- 卡片：`.card` + `.card-title` + `.card-subtitle`
- 按钮：
  - 主按钮：`.btn-primary`
  - 次按钮：`.btn-secondary`
  - 危险按钮：`.btn-danger`
- 标签：`.chip` / `.chip-active`
- 文本工具类：`.muted` / `.text-xs`
- 布局工具类：`.row` / `.row-between` / `.stack-*` / `.gap-12`

## 4. 动效规范
- 进入动效：`.motion-enter`
- 延迟节奏：`.delay-1` / `.delay-2` / `.delay-3`
- 动效原则：只做入场与层次引导，不做复杂过场，保证性能稳定。

## 5. 页面接入现状
- 已接入：登录、资料编辑、推荐、会话列表、聊天、我的页
- 未接入：后续新增页面需优先复用全局类，避免重复定义颜色与按钮样式

## 6. 下一步建议
- 增加状态颜色 token（成功/警告/错误）并统一 Toast 风格
- 增加深浅主题开关（如果后续业务需要）
- 补充 Figma 页面与代码类名映射，形成设计到开发的单一标准
