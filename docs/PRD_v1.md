# 扩列社交小程序 PRD v1（MVP）

本文档描述 MVP 目标范围与执行计划，是需求基线文档。  
实现状态请结合 [功能实现清单](IMPLEMENTED_FEATURES.md) 查看；后续阶段请结合 [产品路线图](ROADMAP.md) 查看。

## 1. 项目信息
- 产品代号：扩列星球（暂定）
- 目标版本：MVP v1.0
- 目标周期：4-6 周（本文件附 2 周首冲刺拆解）
- 平台：微信小程序 + 后端 API + 管理后台（最小版）

## 2. 版本目标
- 让新用户在 3 分钟内完成：登录 -> 完善资料 -> 获得推荐 -> 发出首条消息。
- 验证核心闭环：
  - 有人可匹配
  - 愿意发起聊天
  - 遇到风险可举报并得到处理

## 3. 用户与场景
- 目标用户：18-30 岁，学生/职场新人，想通过兴趣扩列认识新朋友。
- 核心场景：
  - 按兴趣扩列（摄影、游戏、运动等）
  - 同城认识新朋友
  - 匹配后快速破冰聊天

## 4. 核心指标（上线后 30 天）
- D1 留存 >= 25%
- 资料完善率 >= 70%
- 匹配后 24h 首条消息发送率 >= 40%
- 举报处理时效 <= 24h

## 5. 功能范围
### 5.1 P0（MVP 必须）
- 账号体系：微信授权登录，首次登录自动建档。
- 资料系统：头像、昵称、性别、年龄段、城市、3 个兴趣标签、个性签名。
- 推荐与匹配：按兴趣+城市推荐，支持喜欢/跳过，双向喜欢即匹配。
- 聊天系统：匹配后 1v1 文本聊天，会话列表+未读红点。
- 风控能力：举报用户/消息、拉黑、敏感词基础拦截。
- 合规能力：用户协议、隐私政策、账号注销、举报入口。
- 管理端最小能力：举报队列、封禁/解封、处理记录。

### 5.2 P1（后置）
- 语音房、动态广场、群聊、会员体系、实名认证、复杂推荐算法。

## 6. 页面清单（小程序）
- 登录页
- 资料创建/编辑页
- 推荐列表页
- 个人详情页（他人）
- 匹配成功页
- 会话列表页
- 聊天页
- 我的页
- 设置页（协议/隐私/黑名单/注销）
- 举报页

## 7. 关键流程
1. 用户登录并同意协议。
2. 用户完成资料并至少选择 3 个兴趣标签。
3. 系统返回推荐用户列表。
4. 用户执行喜欢/跳过；双方喜欢则创建匹配。
5. 匹配成功后进入会话并发送消息。
6. 任一方可举报或拉黑，进入后台审核闭环。

## 8. 数据模型（MVP）
- `users`：id, openid, nickname, avatar, gender, age_range, city, bio, status, created_at
- `user_tags`：user_id, tag_id
- `swipes`：from_user_id, to_user_id, action(like/pass), created_at
- `matches`：id, user_a, user_b, status(active/closed), created_at
- `conversations`：id, match_id, last_message_at
- `messages`：id, conversation_id, sender_id, type(text), content, created_at
- `reports`：id, reporter_id, target_user_id, target_message_id, reason, status, created_at
- `blocks`：user_id, blocked_user_id, created_at

## 9. API 清单（MVP）
- `POST /auth/wx-login`
- `GET /profile/me`
- `PUT /profile/me`
- `GET /recommendations`
- `POST /swipes`
- `GET /matches`
- `GET /conversations`
- `GET /conversations/{id}/messages`
- `POST /conversations/{id}/messages`
- `POST /reports`
- `POST /blocks`
- `DELETE /blocks/{blockedUserId}`

## 10. 非功能要求
- 性能：推荐接口 P95 < 500ms；发消息接口 P95 < 300ms。
- 安全：统一鉴权、敏感词过滤、行为日志留存 90 天。
- 稳定性：核心 API 可用性 >= 99.9%（MVP 阶段目标）。
- 合规：具备协议、隐私、注销、举报、未成年人说明。

## 11. 验收标准（上线门槛）
- 从登录到首条消息发送全链路可用。
- 举报、拉黑、封禁流程可闭环可追踪。
- 管理端可查询并处理举报记录。
- 关键埋点可统计：注册、完善资料、喜欢、匹配、首聊、举报。

## 12. 开发任务拆解（前端/后端/测试）
### 12.1 前端（小程序）
- FE-01：登录鉴权页与登录态管理。
- FE-02：资料创建/编辑页（字段校验 + 标签选择）。
- FE-03：推荐列表 + 详情页 + 喜欢/跳过交互。
- FE-04：匹配成功页 + 会话列表 + 聊天页（文本消息）。
- FE-05：设置页（协议/隐私/注销）+ 举报页 + 拉黑能力。
- FE-06：埋点接入（注册、完善资料、喜欢、匹配、首聊、举报）。

### 12.2 后端（API + 管理）
- BE-01：微信登录接口 + JWT 会话机制。
- BE-02：用户资料 CRUD 与标签关系表。
- BE-03：推荐服务（兴趣重合 + 同城优先，含分页）。
- BE-04：喜欢/跳过接口与匹配创建逻辑（幂等处理）。
- BE-05：会话与消息接口（文本消息 + 未读计数）。
- BE-06：举报、拉黑、封禁流程与管理端接口。
- BE-07：敏感词过滤中间件 + 审计日志。

### 12.3 测试（QA）
- QA-01：登录/注册/资料链路功能测试。
- QA-02：匹配与聊天链路功能 + 异常流测试（被拉黑、被封禁）。
- QA-03：举报与后台处理闭环测试。
- QA-04：接口压测（推荐、发消息）与基础安全测试（越权、重放）。
- QA-05：回归测试与上线冒烟清单。

## 13. 两周冲刺排期（示例）
### Week 1
- Day 1-2：技术方案冻结、库表创建、鉴权与基础框架搭建。
- Day 3-4：资料模块 + 推荐模块（前后端联调）。
- Day 5：喜欢/匹配逻辑联调，完成核心闭环第一版。

### Week 2
- Day 1-2：会话与聊天模块（文本）联调。
- Day 3：举报/拉黑/封禁 + 管理端处理页。
- Day 4：埋点、性能优化、缺陷修复。
- Day 5：全链路回归、上线检查、预发布验收。

## 14. 风险与对策
- 冷启动无用户：首版准备机器人种子池或白名单内测用户。
- 审核压力：先上基础词库 + 人工复核机制。
- 匹配质量不足：MVP 阶段快速迭代规则，记录点击与会话反馈。
